# 보드게임 허브 — 설계 문서

## 프로젝트 개요
- 이름: BoardHub (보드게임 허브)
- 주요 게임: 마이티 (추후 확장 가능)
- 목적: 포트폴리오 + 실제 플레이 가능한 게임 사이트
- URL 구조: / (허브) → /games/mighty (로비) → /games/mighty/[roomId] (게임)

---

## 기술 스택

### Monorepo
- Turborepo (apps + packages 구조)
- pnpm workspaces

### Frontend (apps/web)
- Next.js 14 (App Router)
- TypeScript 5
- Tailwind CSS 3
- Framer Motion 11 (카드 애니메이션, 페이지 전환)
- Zustand (클라이언트 게임 상태)
- Socket.io-client (실시간 통신)
- React Three Fiber (3D 카드 효과)
- tsparticles (승리 파티클 효과)
- shadcn/ui (기본 컴포넌트)

### Backend (apps/server)
- NestJS 10
- Socket.io (WebSocket Gateway)
- TypeScript 5
- Supabase (Auth + PostgreSQL)
- @supabase/supabase-js

### 공유 패키지 (packages/game-engine)
- 순수 TypeScript (의존성 없음)
- 마이티 게임 로직 전체
- FE/BE 양쪽에서 import

### Phase 2 — 게임 엔진 (packages/game-engine)
**목표:** 마이티 게임 로직 100% 구현 + 테스트

#### 2-1. 타입 정의 (types.ts)

```typescript
type Suit = 'spade' | 'heart' | 'diamond' | 'club'
type Rank = 'A' | 'K' | 'Q' | 'J' | '10' | '9' | '8' | '7' | '6' | '5' | '4' | '3' | '2'

interface Card {
  suit: Suit | 'joker'
  rank: Rank | 'joker'
  id: string  // 'spade-A', 'joker' 등
}

interface Bid {
  playerId: string
  count: number               // 13~20
  trump: Suit | 'no-trump' | null   // null = pass
}

type GamePhase =
  | 'waiting'
  | 'dealing'
  | 'bidding'
  | 'kitty'
  | 'friend-select'
  | 'playing'
  | 'finished'

interface Player {
  id: string
  name: string
  hand: Card[]
  isAI: boolean
  aiLevel?: 'beginner' | 'intermediate' | 'expert'
}

interface TrickCard {
  playerId: string
  card: Card
}

interface Trick {
  cards: TrickCard[]
  winnerId: string
  points: number
}

interface GameState {
  phase: GamePhase
  players: Player[]
  kitty: Card[]               // 주공 교환용 3장
  currentBid: Bid | null
  trump: Suit | 'no-trump' | null
  friendCard: Card | null
  presidentId: string | null  // 주공
  friendId: string | null     // 파트너 (공개 전 null)
  currentTurnId: string | null
  tricks: Trick[]
  currentTrick: TrickCard[]
  trickCount: number          // 0~10
  pointsScored: number        // 주공팀 점수
  winner: 'president' | 'opposition' | null
}
```

#### 2-2. 카드 덱 (deck.ts)
- 53장 생성 (52 + 조커)
- Fisher-Yates 셔플
- 10장씩 5명 배분, 나머지 3장 키티

#### 2-3. 입찰 (bidding.ts)
- 유효 입찰 검증 (이전보다 높아야 함)
- 노트럼프는 같은 count에서 일반 트럼프보다 강함 (노트럼프 13 > 트럼프 13)
- 전원 패스 시 마지막 입찰자 주공 (또는 재딜)
- 최대 공약: 20

#### 2-4. 트릭 (trick.ts)

핵심 판정 우선순위:
```
1. 마이티 (♠A, 트럼프가 스페이드이면 ♥A)
2. 조커 (선이 아닐 때)
3. 트럼프 수트 (높은 랭크 우선)
4. 선 수트 (높은 랭크 우선)
5. 나머지
```

특수 규칙:
- **팔로우**: 같은 수트 있으면 반드시 따라야 함
- **조커콜**: 선이 조커를 낼 때 원하는 수트 선언
- **조커 제한**: 첫 트릭/마지막 트릭에 선으로 조커 불가
- **프렌드 공개**: 프렌드 카드를 내는 순간 파트너 공개

#### 2-5. 점수 계산 (scoring.ts)
- 점수 카드: A(1점), 10(1점), K(1점) → 수트당 3점 × 4수트 = 12점
- 조커 포함 트릭: +1점 → 최대 13점
- 주공팀 승리: 획득 점수 ≥ 선언 공약
- 야당팀 승리: 주공팀 공약 미달

#### 2-6. AI (ai/)

**초보 (beginner.ts)**
- 유효 카드 중 랜덤 선택
- 입찰: 패가 좋아도 70% 확률로 패스

**중수 (intermediate.ts)**
- 점수 카드(A, 10, K) 잡으려는 경향
- 강한 카드(마이티, 트럼프 높은 것) 아끼기
- 파트너 플레이어 식별 후 도움
- 트릭 이길 수 없으면 약한 카드 처리

**고수 (expert.ts) — MCTS**
```
알고리즘: Information Set MCTS (IS-MCTS)

1. Determinization (50회)
   - 상대 카드를 남은 카드에서 확률적으로 샘플링
   - 각 샘플에서 완전정보 게임 생성

2. MCTS per determinization
   - Selection: UCB1 = (wins/n) + C × √(ln(N)/n), C=1.41
   - Expansion: 미방문 유효 수 확장
   - Simulation: 랜덤 플레이아웃 (끝까지)
   - Backpropagation: 승/패 결과 역전파

3. 최종 선택
   - 50회 시뮬레이션 결과 집계
   - 가장 높은 평균 승률의 수 선택
   - 제한시간: 1000ms
```

완료 기준: 모든 단위 테스트 통과 (`pnpm test`)

### Phase 3 — 백엔드 (apps/server)
**목표:** REST API + Socket.io 게임 서버 완성

#### 3-1. Supabase DB 스키마

```sql
-- 유저 프로필
create table users (
  id uuid references auth.users primary key,
  username text unique not null,
  avatar_url text,
  created_at timestamptz default now()
);

-- 게임 전적
create table game_records (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references users(id),
  game_type text default 'mighty',
  role text check (role in ('president', 'friend', 'opposition')),
  result text check (result in ('win', 'lose')),
  score integer,
  bid_count integer,
  trump text,
  created_at timestamptz default now()
);

-- RLS 활성화
alter table users enable row level security;
alter table game_records enable row level security;
```

#### 3-2. 방 관리 REST API

```
POST   /rooms          방 생성 { name, isPrivate, password? }
GET    /rooms          공개 방 목록
GET    /rooms/:id      방 상세
DELETE /rooms/:id      방 삭제 (방장만)
```

#### 3-3. Socket.io 이벤트

**클라이언트 → 서버**
```typescript
'join-room'      { roomId: string, userId: string }
'leave-room'     { roomId: string }
'ready'          { roomId: string }
'place-bid'      { roomId: string, bid: Bid }
'pass-bid'       { roomId: string }
'exchange-kitty' { roomId: string, discardCards: Card[] }
'select-trump'   { roomId: string, trump: Suit | 'no-trump' }
'select-friend'  { roomId: string, friendCard: Card }
'play-card'      { roomId: string, card: Card }
'joker-call'     { roomId: string, suit: Suit }
```

**서버 → 클라이언트**
```typescript
'game-state'    GameState           // 전체 상태 (자기 패 포함)
'your-turn'     { phase: GamePhase, validCards: Card[] }
'player-joined' { player: Player }
'player-left'   { playerId: string }
'game-started'  { hand: Card[] }    // 각자 자기 패만
'bid-placed'    { playerId: string, bid: Bid }
'card-played'   { playerId: string, card: Card }
'trick-ended'   { winnerId: string, trick: Trick }
'friend-revealed' { friendId: string }
'game-ended'    { result: 'president' | 'opposition', scores: Record<string, number> }
'error'         { message: string }
```

#### 3-4. 게임 서버 흐름

```
[방 생성] → [5명 입장] → [모두 ready]
    ↓
[카드 배분 (10장씩 + 키티 3장)]
    ↓
[입찰 라운드] → [주공 확정]
    ↓
[주공: 키티 3장 보기 → 3장 버리기 → 트럼프 선언]
    ↓
[프렌드 카드 선언]
    ↓
[트릭 진행 × 10회]
  - 선 플레이어 카드 선택
  - 순서대로 5명 카드 제출
  - 트릭 승자 계산
  - 다음 트릭 선 = 이전 트릭 승자
    ↓
[최종 점수 계산] → [결과 저장 (Supabase)] → [재시작 여부]
```

완료 기준: Postman REST 테스트 + wscat 소켓 연결 확인

```typescript
// 입찰: 패 강도 평가 후 적정 공약 계산

//   2. 야당팀이면: 주공팀 트릭 방해

```

### 고수 AI — MCTS (expert.ts)
```
IS-MCTS (Information Set Monte Carlo Tree Search)

파라미터:
- numDeterminizations: 50   (샘플링 횟수)
- numSimulations: 20        (샘플당 시뮬레이션)
- explorationConstant: 1.41 (UCB1 C값)
- timeLimit: 1000           (ms)

알고리즘:
for i in range(numDeterminizations):
  1. 상대 패를 남은 카드에서 랜덤 샘플링 (Determinize)
  2. MCTS 트리 탐색:
     - Selection: UCB1으로 최선 노드 선택
     - Expansion: 미방문 유효 수 추가
     - Simulation: 끝까지 랜덤 플레이
     - Backpropagation: 승패 역전파
  3. 각 수의 누적 승률 기록

최종: 누적 승률 가장 높은 수 반환
```
